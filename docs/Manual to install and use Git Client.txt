Per lavorare ai progetti c'è bisogno di installare Git client sul pc.

Per Linux e Mac seguire questa guida:
https://git-scm.com/book/en/v2/Getting-Started-Installing-Git
Per il resto la parte della chiave dovrebbe essere uguale a Windows (a parte qualche comando, che ho specificato sotto)

Per Windows:
Andare su https://git-scm.com/download/win and "click here to download manually", e installare il tutto.

Dopo c'è bisogno di creare e utilizzare una chiave per l'shh.
Aprire il Git Bash e controllare se c'è una versione di ssh con il comando: ssh -V
Creare la chiave con il comando:  ssh-keygen -o -t rsa -C "email@example.com" -b 4096  (inserire l'email di iscrizione a GitLab)
Aggiungere la chiave ssh all'account GitLab:  Windows -> cat ~/.ssh/id_rsa.pub | clip
                                              macOS ---> pbcopy < ~/.ssh/id_ed25519.pub
                                              Linux ---> xclip -sel clip < ~/.ssh/id_ed25519.pub   //requires the xclip package
                                              
Ora bisogna aggiungere anche manualmente la chiave sul sito di GitLab:
    1.Andare sul proprio account
    2.Settings
    3.SSH Keys
    4.Incollare nel text-box la public key presa in: cat ~/.ssh/id_rsa.pub (inizia con ssh-ed25519 oppure ssh-rsa e finisce con la propria email)
    5.Dare un titolo.
    6.CLiccare Add key

Ora possiamo clonare la respository (nella directory che preferite, potete spostarvi nelle directories come in linux):
    1.Con il comando: git clone http://aminsep.disi.unibo.it/gitlab/ToMarco/twitter-tracker.git
    2.Inserite Username e Password di GitLab

Ora il resto dei comandi (push/pull/commit) sono come GitHub:
    Caricare un NUOVO file creato localmente o per file spostati di cartelle: git add filename / git add . ("." aggiunge tutti in nuovi file->molto più comodo)
    Scaricare: git pull //farlo ogni volta che volete lavorare al progetto
    Commit e Caricare: 1. git commit -m "Commit text"  //RICORDARE: PRIMA DI FARE UNA PUSH BISOGNA SEMPRE FARE IL COMMIT ALTRIMENTI NON VE LO FA FARE
                       2. git push
                       
                       
   Forse (probabilmente) la prima volta vi darà errore e vi dirà che dovreste fare la Merge, dovrebbe aprirsi Visual Studio code (se lo usate) -> poi chiudetelo e dovrebbe risolversi
   da solo, altrimenti fate: "git merge" e poi riprovate.
